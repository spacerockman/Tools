---
description: Standard loop for debugging and fixing issues
---
1. Read the error message or issue description carefully.
2. Search for relevant code using `grep_search`.
3. Use `list_dir` and `view_file` to understand the context.
4. Apply the fix using `replace_file_content` or `multi_replace_file_content`.
5. Verify the fix by running relevant scripts or checking the backend logs.
6. (If UI) Use the browser tool to confirm visual fixes.
